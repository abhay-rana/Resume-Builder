# Skills Section Best Practices

> **Quick Link:** Build your skills-optimized resume at [ResumeFreePro.com](https://www.resumefreepro.com)

Your skills section is one of the first things recruiters and ATS systems scan. Here's how to make it count.

---

## Skills Section Placement

### Where to Put Skills

| Resume Type | Skills Placement |
|-------------|-----------------|
| Technical roles | Near top, after summary |
| Entry-level | After education |
| Senior roles | After experience |
| Career change | Near top to highlight transferable skills |

---

## Types of Skills to Include

### Hard Skills (Technical)

Specific, teachable abilities:

- Programming languages (Python, JavaScript)
- Software proficiency (Salesforce, Adobe Suite)
- Data analysis (SQL, Excel, Tableau)
- Certifications (PMP, AWS, CPA)
- Languages (Spanish - Fluent)
- Tools (Git, Jira, Figma)

### Soft Skills (Interpersonal)

Workplace and people skills:

- Leadership
- Communication
- Problem-solving
- Team collaboration
- Time management
- Adaptability
- Critical thinking
- Conflict resolution

## How to Format Skills

### Option 1: Categorized List (Recommended)

```
SKILLS

Technical: Python, JavaScript, React, Node.js, PostgreSQL, AWS, Docker
Tools: Git, Jira, Figma, VS Code, Postman
Methodologies: Agile, Scrum, TDD, CI/CD
Languages: English (Native), Spanish (Professional)
```

### Option 2: Simple List

```
SKILLS

Python | JavaScript | React | AWS | Docker | Git | Agile | SQL
```

### Option 3: Proficiency Levels (Use Carefully)

```
SKILLS

Expert: Python, JavaScript, React
Proficient: AWS, Docker, PostgreSQL
Familiar: Go, Kubernetes
```

**Warning:** Avoid skill bars or percentages. They're subjective and ATS can't read them.

## Skills to AVOID Listing

### Don't Include:

| Skip This | Why |
|-----------|-----|
| Microsoft Office | Assumed for most roles |
| "Computer skills" | Too vague |
| Email | Obviously required |
| Typing | Unless data entry role |
| Social media | Unless marketing role |
| Soft skills only | Need hard skills too |
| Outdated tech | Shows you're not current |

### Be Careful With:

- **Self-assessed ratings** - Subjective and often inflated
- **Every skill you've touched** - Focus on relevant ones
- **Buzzwords without substance** - "Synergy", "Best practices"

## Tailoring Skills for Each Job

### Step-by-Step Process:

1. **Read the job description** carefully
2. **Highlight required skills** they mention
3. **Match your skills** to their requirements
4. **Use their exact terminology** (if true)
5. **Order by relevance** to the role

### Example:

**Job requires:** Python, SQL, data visualization, communication

**Your skills (reordered):**
```
Technical: Python, SQL, Tableau, Power BI, Excel
Additional: Data visualization, Statistical analysis, Stakeholder communication
```

## Skills Section Length

### Guidelines:

| Experience Level | Number of Skills |
|-----------------|------------------|
| Entry-level | 8-12 skills |
| Mid-level | 12-18 skills |
| Senior | 15-25 skills |

### Quality Over Quantity:

- Only list skills you can discuss in an interview
- Be prepared to demonstrate technical skills
- Remove skills you haven't used in 2+ years

---

## Common Mistakes

### 1. Listing Everything

**Bad:** 50 skills including obvious ones
**Good:** 15-20 highly relevant skills

### 2. No Organization

**Bad:** Random list of unrelated skills
**Good:** Grouped by category (Technical, Tools, Soft Skills)

### 3. Lying About Proficiency

**Bad:** Claiming "Expert" in something you've used once
**Good:** Honest assessment of your abilities

### 4. Ignoring the Job Description

**Bad:** Generic skills section for every application
**Good:** Tailored skills matching job requirements

### 5. Using Skill Bars

**Bad:** Visual skill ratings (can't be read by ATS)
**Good:** Text-based proficiency levels if needed

---

## Skills Section Checklist

Before submitting:

- [ ] Skills are relevant to target role
- [ ] Hard skills and soft skills balanced
- [ ] Keywords from job description included
- [ ] Organized in clear categories
- [ ] No obvious/outdated skills
- [ ] Can discuss each skill in interview
- [ ] No visual elements (bars, charts)
