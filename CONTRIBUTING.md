# Contributing to ResumeFreePro

Thank you for your interest in contributing to ResumeFreePro! We welcome contributions from the community.

## Ways to Contribute

### :bug: Report Bugs
Found a bug? Please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Browser and OS information

### :bulb: Suggest Features
Have an idea? We'd love to hear it! Open an issue with:
- Feature description
- Use case / problem it solves
- Any mockups or examples

### :art: Submit Template Designs
Want to create a new resume template? Awesome! Please:
- Follow our design guidelines
- Ensure ATS compatibility
- Include preview images
- Test across different content lengths

### :globe_with_meridians: Add Translations
Help us reach more users worldwide:
- Check existing translations first
- Follow our i18n format
- Include all UI strings

### :book: Improve Documentation
Documentation improvements are always welcome:
- Fix typos or unclear wording
- Add missing information
- Improve examples

---

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a new branch for your changes
4. **Make** your changes
5. **Test** thoroughly
6. **Commit** with clear messages
7. **Push** to your fork
8. **Open** a Pull Request

---

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/resume-builder.git

# Navigate to project
cd resume-builder

# Install dependencies
yarn install

# Start development server
yarn dev
```

---

## Code Guidelines

### Commit Messages
Use clear, descriptive commit messages:
```
feat: add new template option
fix: resolve PDF export issue on Safari
docs: update installation guide
style: improve button hover states
```

### Code Style
- Use TypeScript
- Follow existing code patterns
- Add comments for complex logic
- Write meaningful variable names

### Testing
- Test your changes thoroughly
- Check cross-browser compatibility
- Verify mobile responsiveness

---

## Pull Request Process

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure all tests pass
4. Update CHANGELOG if applicable
5. Request review from maintainers

### PR Title Format
```
[feat] Add new executive template
[fix] Resolve font loading issue
[docs] Update README with new features
```
